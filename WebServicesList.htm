Authenticating Web Service Requests

In addtion to each request's input fields, the request must be authenticated by one of the following two methods:

Method 1: Provide CORRIDOR user credentials for each request

With this method, the user credentials must be specified in each individual request sent to the web service. This method is preferred if only sending a single request.

The additional fields that must be provided in the request include:

Name Notes/Description
AliasName The CORRIDOR database alias name.
LoginID The CORRIDOR user name.
LoginPassword The CORRIDOR user's password.

Method 2: Use the Session web service to establish a user session

With this method, the Session web service is used to establish a session with CWS. This is the preferred method if making a batch of requests to CWS. Performance gains are very substantial over providing credentials with each request (method 1).

NOTE: Sessions expire after a configurable amount of time.

Using this method, the only additional field required to authenticate other requests is:

Name Notes/Description
SessionToken Returned from the Session service's Login method (see below).

The steps in using this method are:

  1. Establish the session by calling the Login method of the Session web service. Store the session token returned.
  2. Call multiple other API requests, providing the session token as the credentials.
  3. End the session by calling the Logout method of the Session web service.

Session Login Request

Given user credentials, returns a token that can be used for authenticating subsequent requests.

Request Data Fields
Name Notes/Description
AliasName The CORRIDOR database alias name.
LoginID The CORRIDOR user name.
LoginPassword The CORRIDOR user's password.
Response Data Fields
Name Notes/Description
SessionToken The token that can be used to authenticate subsequent requests.
StatusMessage Success/Failed.

Session Logout Request

Given the session token returned from the Login request, ends the user session.

Request Data Fields
Name Notes/Description
SessionToken The token identifying the session to be ended. The session token will become invalid.
Response Data Fields
Name Notes/Description
StatusMessage Success/Failed.

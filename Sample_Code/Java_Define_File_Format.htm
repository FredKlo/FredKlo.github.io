Java Define File Format

Like C++ defines header files corresponding java defines are in the following files. Below is an example of the WorkOrderDefines.java file. All Java define files are written with well formed comments that are extracted to a corridor-api-javadocs file.

 

package com.continuum.api;

/**

   The IWorkOrderDefines interface contains the schema tag names for

   accessing Work Order data. <br>

*/

public interface IWorkOrderDefines

{

    // Supported Request Types

 * in place of the path map data.

 */

String ADD_WO_LABOR_REQUEST        = "AddWorkOrderLaborRequest";

/**

   The add request data for the Work Order Labor API must include a path map data.

   The update request data must either include WOLaborKey data or a path map data

   if the key is not used. The move request data must include a from path and a to

   path map data. <br><br>

   The path map should contains the Work#, Item#, Squawk# and zero or more Step#s. For the

   constants for the path tag names see <a href="ICommonDefines.html">ICommonDefines</a>

   E.g. for the Add or Update Work Order Labor request the RITA_DATA_PATH  holds the

   path info. For Move Work Order Labor request the RITA_DATA_FROM_PATH and

   RITA_DATA_TO_PATH are defined to hold the source and destination info.

   The data contained in the path maps are defined under

   <a href="IWorkOrderDefines.Path.html">IWorkOrderDefines.Path</a>

   E.g. WO_NUMBER, WO_ITEM_NUMBER, etc.

   As an example, the request data structure for Add request will be like:

   <pre>

       RequestMap =

       [

         {RITA_DATA_PATH, &ltPathMap&gt},

         {WO_LABOR_PERFORMED_BY, "username"},

         {WO_LABOR_COMMENTS, "some comments"},

         {etc., etc.}

       ]

       where &ltPathMap&gt is another Map as,

       PathMap =

       [

         {WO_NUMBER, WorOrder#},

         {WO_ITEM_NUMBER, Item#},

         {WO_SQUAWK_NUMBER, Squawk#},

         {WO_STEPS, &ltStepsList&gt}

       ]

       </pre>

   Since the Work Order Labor can be added to a squawk or a step. The WO_STEPS

   entry in the PathMap is optional. But if add labor to one or more steps then

   it must be provided in an list;  so &ltStepsList&gt is a list of steps as, <br>

   StepsList = [Step1#, Step2#,...] <br>

   <br>

   Use java.util.Map and java.util.List to build this structure.

*/

    public interface WorkOrderLabor

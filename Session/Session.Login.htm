Login

    Description

      The request creates a CORRIDOR API session that can be reused to validate future requests. The session will remain active for 60 minutes(This time can be altered in the web service config file) since last use before it has to be regenerated. All Corridor web services require login information. Either the session token or the required fields for the login web service can be used as the login parameters.

Core Methods

 There are only 3 methods to interface with the Corridor APIs. The signature of these methods, declared in Rita.h file, are:

 

  • BSTR CORRIDOR_API_MYLIB_DECL StartNewSession (BSTR sDbAlias, BSTR sUserName, BSTR sPassword, BSTR sSystemID, BSTR& sErrorMessage)

 

The StartNewSession method enables the starting of a new session. It takes the database name alias as listed under the database drop down menu under the Corridor login screen.. The sUserName and the sPassword parameters are used for user login and the sSystemId is the parameter for workstation/server identity. The sErrorMessage parameter contains any errors encountered during the initiation of the session. On the successful initiation of a session, the method returns a session id; otherwise an INVALID_SESSION id is returned. The session id is the handle to the session and is required for processing a request and ending the session. The BSTR is a 32-bit character pointer.

 

  • bool CORRIDOR_API_MYLIB_DECL EndSession (const char* sSessionID)

 

The EndSession method destroys the session. It takes the sSessionID, the session handle, returned by the StartNewSession method.

 

  • bool  CORRIDOR_API_MYLIB_DECL ProcessRequest  (const char* sSessionID, IRitaRequest& Request, IRitaResponse& Response)

 

The ProcessRequest takes the session handle, sSessionID, returned by the StartNewSession method, a request object, Request, and a response object Response, to process the request and return the results in the response object. For an add/update request the modifications are committed if the request is processed successfully; otherwise it is rolled back effectively erasing any errors encountered during processing.
